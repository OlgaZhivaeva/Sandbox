# Многопользовательский чат на сокетах
Проект представляет три варианта многопользовательского консольного чата.

##  Блокирующий сервер

Простой TCP-сервер, который:
- Создаёт сокет.
- Привязывается к адресу 127.0.0.1 и выбранному порту.
- Ожидает подключения клиента.
- Принимает сообщения и отправляет их обратно.
- Корректно обрабатывает отключение клиента.

### Запуск

Запустите сервер
```shell
python blocking_server.py
```

В другом терминале подключитесь к нему с помощью `telnet`
```shell
telnet 127.0.0.1 8000
```
Сервер будет дублировать все набранные символы.

### Как это работает

Сервер обслуживает клиентов поочереди. В коде две блокирующие операции 
`serv_sock.accept()` и `conn.recv(1024)`.

`serv_sock.accept()` блокирует выполнение программы, пока какой-нибудь клиент не подключится. 

`conn.recv(1024)` блокирует выполнение программы пока не получит данные от подключившегося клиента.

Сервер не может обслуживать двух клиентов одновременно из-за блокирующего вызова `conn.recv(1024)` внутри внутреннего цикла `while True`. 
Поток выполнения Python останавливается на этой строке и ждет, пока от первого клиента придут данные или соединение будет разорвано.
Подключение второго клиента возможно только после завершения внутреннего цикла `while True` и закрытия соединения с первым клиентом.

Пока сервер обслуживает клиента, новые клиенты при попытке подключения будут поставлены в очередь на уровне операционной системы. 
И как только сервер будет освобожден соединение первого в очереди клиента будет принято сервером.

## Чат с использованием select

TCP-сервер, обслуживающий несколько клиентов через select.select(), который:

- Работает в одном потоке.
- Переводит серверный и клиентские сокеты в неблокирующий режим.
- Хранит список подключённых клиентов.
- Через select.select() определяет сокеты, готовые к чтению.
- Принимает новые подключения.
- Получает сообщения клиентов.
- Рассылает сообщения всем остальным клиентам.
- Удаляет отключившихся клиентов.
- Не завершается из-за отключения одного клиента.

### Запуск

Запустите сервер
```shell
python select_server.py
```

В других терминалах подключитесь к нему с помощью `telnet`
```shell
telnet 127.0.0.1 8000
```
В окне `telnet` напишите сообщение и отправьте его нажав клавишу Enter. 
Сообщение будет отправлено всем подключившимся клиентам.

### Особенности работы

В отличие от блокирующего сервера, вызовы `sock.accept()` и `sock.recv(1024)` являются не блокирующими. 
Для перевода сокета в этот режим используется метод `socket.setblocking(False)` 
Неблокирующий вызов возвращает управление немедленно, либо с данными, либо с ошибкой. 
Если данных пока нет неблокирующий вызов recv(1024) вернёт ошибку BlockingIOError, 

Вызов select.select() возвращает список сокетов готовых к чтению. Сокет готов к чтению означает, что 
во входящем системном буфере этого сокета появились данные (байты), которые можно забрать вызовом recv(), 
либо соединение было закрыто(в этом случае recv вернет 0 байт).

Для корректной отправки сообщений используется входной буфер(словарь `buffers`). Буфер нужен, чтобы накапливать входящий поток байтов, 
пока в нем не наберется целое сообщение (до символа \n).

Если одно сообщение придет двумя recv:

При первом вызове recv код заберет первую часть сообщения и положит его во входной буфер клиента (buffers[conn]). 
Он проверит, есть ли в буфере символ переноса строки \n. Так как его там нет, код не будет ничего отправлять в чат, 
а просто закончит работу и будет ждать следующих данных.

При втором вызове recv код заберет вторую часть сообщения с \n и прибавит его к буферу. В буфере получится склеенная строка. 
Код снова проверит буфер на наличие \n, теперь он его найдет, разрежет строку, достанет целое сообщение, отправит его в чат.

Если два сообщения пришли одним recv:

Код заберет всю эту строку за один раз и запишет её в буфер клиента. Он обнаружит в буфере символ \n и с помощью 
метода split разрежет строку по этому символу. В результате получится список, состоящий из двух сообщений.
Код запустит цикл по этому списку и по очереди обработает каждое сообщение.

## Чат с использованием selectors

TCP-сервер, способный одновременно обслуживать несколько клиентов в одном потоке выполнения, который:
- Получает готовые события через selector.select().
- Определяет, какому сокету принадлежит событие.
- Вызывает соответствующий обработчик.
- Возвращается к ожиданию следующих событий.

Многопоточный клиент для участия в чате, который может:
- подключаться к серверу;
- запрашивать имя пользователя;
- отправлять введённые сообщения;
- показывать сообщения других участников;
- завершать работу по команде /quit.

### Запуск

Запустите сервер
```shell
python selector_server.py
```
В других терминалах запустите клиентскую программу
```shell
python client.py
```
После запуска клиент попросит ввести ваше имя. 
Для выхода введите команду /quit.

### Особенности работы

Использование selectors позволяет обслуживать много клиентов в одном потоке. Вместо того чтобы 
блокироваться на каждом сокете отдельно, сервер регистрирует все сокеты в одном системном селекторе.
Вызов selector.select() временно приостанавливает поток. ОС сама будит поток, как только на любом из 
сокетов происходит событие (пришли данные или новое подключение). Получив список готовых сокетов, 
поток сервера мгновенно обрабатывает их по очереди. 

На сервере потоки запрещены. Каждый поток в ОС требует памяти под стек, если подключится 
10 тысяч клиентов, серверу просто не хватит оперативной памяти. Когда потоков слишком много, процессор 
тратит больше времени на переключение между ними, чем на полезную работу. Потоки разделяют общую память. 
Возникает риск  состояний гонки (Race Conditions), требующих блокировок (Mutex/Locks), что может приводить
к взаимным блокировкам (Deadlocks).

В клиенте потоки разрешены и удобны. Клиент обслуживает ровно одно сетевое соединение и одного пользователя. 
Ему не нужно обслуживать тысячи подключений. Клиенту нужно делать две вещи одновременно: 
ждать ввода от пользователя (блокирующий input()) и ждать сообщений от сервера (блокирующий recv()).


В отличие от прямого вызова select.select(), selectors — это высокоуровневая абстракция. 
DefaultSelector автоматически выбирает самый быстрый механизм для конкретной ОС (epoll на Linux, kqueue на macOS) 
и позволяет удобно привязывать callback-функции к событиям на сокетах при помощи selector.register().
Метод register(conn, events, data) позволяет прикрепить к сокету любые данные(в данном случае это callback-функции 
read_client или accept_client). Это удобно, потому что при наступлении события не нужно писать ветвления if для поиска нужной функции.

Цикл while True с вызовом selector.select() и последующим выполнением callback-функций — 
это упрощенная реализация Event Loop, который лежит в основе asyncio.
