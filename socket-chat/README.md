# Многопользовательский чат на сокетах

##  Блокирующий сервер

Простой TCP-сервер, который:
- Создаёт сокет.
- Привязывается к адресу 127.0.0.1 и выбранному порту.
- Ожидает подключения клиента.
- Принимает сообщения и отправляет их обратно.
- Корректно обрабатывает отключение клиента.

### Запуск

Запустите сервер
```shell
python blocking-server.py
```

В другом терминале подключитесь к нему с помощью `telnet`
```shell
telnet 127.0.0.1 8000
```
Сервер будет дублировать все набранные символы.

### Как это работает

Сервер обслуживает клиентов поочереди. В коде две блокирующие операции 
`serv_sock.accept()` и `conn.recv(1024)`.

`serv_sock.accept()` блокирует выполнение программы, пока какой-нибудь клиент не подключится. 

`conn.recv(1024)` блокирует выполнение программы пока не получит данные от подключившегося клиента.

Пока сервер обслуживает клиента, новые клиенты при попытке подключения будут поставлены в очередь на уровне операционной системы. 
И как только сервер будет освобожден соединение первого в очереди клиента будет принято сервером.

## Чат с использованием select

TCP-сервер, обслуживающий несколько клиентов через select.select(), который:

- Работает в одном потоке.
- Переводит серверный и клиентские сокеты в неблокирующий режим.
- Хранит список подключённых клиентов.
- Через select.select() определяет сокеты, готовые к чтению.
- Принимает новые подключения.
- Получает сообщения клиентов.
- Рассылает сообщения всем остальным клиентам.
- Удаляет отключившихся клиентов.
- Не завершается из-за отключения одного клиента.

### Запуск

Запустите сервер
```shell
python select-server.py
```

В других терминалах подключитесь к нему с помощью `telnet`
```shell
telnet 127.0.0.1 8000
```
В окне `telnet` напишите сообщение и отправьте его нажав клавишу Enter. 
Сообщение будет отправлено всем подключившимся клиентам.
